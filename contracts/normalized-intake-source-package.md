# Contract: Normalized Intake Source Package

- **Status:** Active
- **Canonical Artifact Name:** NormalizedIntakeSourcePackage
- **Current Version:** 1.1.0

## Purpose
The Normalized Intake Source Package is the durable launch-stage input artifact created before AI routing or generation begins. It gives the app one stable shape for pasted text, outlines, photos, PDFs, and uploaded files.

## Producers
- **Entrypoints:**
  - `app/api/homeschool/intake-package/route.ts`
- **Canonical Source Files:**
  - `lib/homeschool/intake/types.ts`
  - `lib/homeschool/intake/service.ts`
  - `components/onboarding/homeschool-onboarding-form.tsx`

## Consumers
- **Entrypoints:**
  - `app/api/homeschool/onboarding/route.ts`
  - `lib/homeschool/onboarding/service.ts`
  - `lib/planning/today-lesson-generation.ts`
- **Processing Logic:**
  - The onboarding flow uses the package's `normalizedText` as the canonical source text for preview, confidence estimation, and first-pass generation.
  - Asset-backed packages expose extraction state and asset metadata without forcing a curriculum source record to exist yet.
  - Downstream app metadata stores a trimmed source-package context projection so later agent calls can recover modality, summary, asset ids, and extraction state without duplicating full asset rows.

## Persistence
- **Storage Location:**
  - `intake_source_packages`
  - `intake_source_assets`
- **Storage Shape:**
  - The package row stores `normalized_text`, `source_fingerprint`, and summary metadata.
  - Asset rows store uploaded file metadata plus asset-level extracted text and extraction status.
  - Curriculum/onboarding metadata may persist a source-package context projection keyed by package id, modality, summary, extraction status, detected chunks, and asset ids.

## Field Definitions

### Required Fields
| Field | Type | Description |
|-------|------|-------------|
| id | string | Stable package identifier. |
| organizationId | string | Household owner of the package. |
| title | string | Parent-facing package title. |
| modality | enum | Launch intake modality: `text`, `outline`, `photo`, `image`, `pdf`, or `file`. |
| status | enum | Package state: `draft`, `ready`, or `failed`. |
| normalizedText | string | Canonical text passed downstream into routing and generation. |
| detectedChunks | string[] | Short extracted lines used in preview. |
| summary | string | Parent-facing package summary for preview. |
| assetCount | number | Number of linked uploaded assets. |
| extractionStatus | enum | Aggregate asset extraction state: `pending`, `ready`, `requires_review`, or `failed`. |
| assets | IntakeSourceAsset[] | Linked uploaded asset records. |
| createdAt | datetime string | Creation timestamp. |
| updatedAt | datetime string | Last update timestamp. |

### Optional Fields
| Field | Type | Description |
|-------|------|-------------|
| learnerId | string | Learner in focus when the package was created. |
| note | string | Parent-authored note attached to the package. |
| sourceFingerprint | string | Deterministic fingerprint of the normalized source content. |

### Derived / Computed Fields
| Field | Source | Logic |
|-------|--------|-------|
| summary | normalizedText + modality + assetCount | Generated by the intake service when the package is created. |
| detectedChunks | normalizedText | Derived from the first normalized non-empty lines. |
| extractionStatus | assets[] | `failed` wins over `requires_review`, which wins over `pending`, which wins over `ready`. |

## Defaults & Fallbacks
- **normalizedText:** Direct text uses the typed value. Asset-backed inputs use extracted text and may append the parent note.
- **image/photo extraction:** Launch-safe fallback is the parent note rather than OCR.
- **pdf extraction:** PDF parsing is attempted first; parent note is the fallback if parsing fails.

## Validation & Invariants
- A package must have usable `normalizedText` before it can be marked `ready`.
- Asset-backed packages must retain the uploaded asset metadata even if extraction falls back to the parent note.
- `assetCount` must match the number of returned asset records.
- `normalizedText` is bounded to the launch-safe ingestion limit.

## Ownership & Hierarchy
- **Parent:** Organization / household
- **Children:** Intake source assets

## Change Impact
- **Downstream Effects:** Shape changes affect onboarding preview, routing, and the first-pass generation pipeline.
- **Related Contracts:** `lesson-draft-artifact.md`, `curriculum-artifact.md`

## Known Gaps / TODOs
- Image and photo OCR are not part of this launch contract yet; note-backed extraction is the current bounded fallback.
- Later phases will add interpretation and bounded-plan contracts that consume this package.
