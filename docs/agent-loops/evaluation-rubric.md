# Evaluation Rubric

Score each category 0–3.

## 1. Input faithfulness
- 0: largely ignores the source
- 1: loosely related but misses key source details
- 2: mostly grounded with some drift
- 3: clearly grounded in the provided material

## 2. Horizon correctness
- 0: expands wildly beyond what the source supports
- 1: somewhat over- or under-expands
- 2: mostly bounded correctly
- 3: exactly matches the available evidence

## 3. Lesson teachability
- 0: parent could not run this lesson
- 1: vague or incomplete
- 2: workable with light editing
- 3: ready to teach as-is

## 4. Parent clarity
- 0: unclear what to do next
- 1: some structure but confusing
- 2: understandable
- 3: obvious and calm

## 5. Activity usefulness
- 0: filler or mismatched
- 1: weakly related or too generic
- 2: useful practice
- 3: clearly extends or reinforces the lesson

## 6. Age / learner fit
- 0: wrong level
- 1: partially mismatched
- 2: mostly appropriate
- 3: clearly tuned to the learner

## 7. Continuity
- 0: tomorrow would make no sense
- 1: weak continuity
- 2: reasonable continuity
- 3: clear next-step coherence

## 8. Pack / interaction fit
- 0: no supported activity shape fits
- 1: pack choice looks weak
- 2: supported shape is acceptable
- 3: pack choice feels strong

## 9. Product wow factor
- 0: no visible value
- 1: some value but not compelling
- 2: meaningful payoff
- 3: obvious “this saved me time” moment

## Required notes

For every scored run, include:
- one sentence on the main success
- one sentence on the main failure
- whether the problem belongs to app, routing, operation, pack, or contract
- whether you would show this output to a beta user
