# Beta Ops Guide

## Purpose

Run the activation beta against the real homeschool wedge and keep support, QA, and review decisions consistent.

## Cohort Rules

- recruit 15 to 25 real-fit households
- tag each household with a cohort label and measurement mode
- avoid broad friend-and-family signups that do not match the wedge

## Weekly Rhythm

- review the event rollup once per week
- log top friction points after every support interaction
- rerun hosted QA on the wedge routes before each scorecard review

## Support Intake

- capture route, household, learner, and exact failure mode
- classify each issue as:
  - blocker
  - rough edge
  - feature request

## Blocker Rules

- a blocker is any issue that breaks onboarding, curriculum creation, Today, learner switching, or records flow for a beta household
- rough edges can stay open only if they do not break the daily operating flow

## Review Output

- create a dated scorecard
- write an explicit go / iterate / no-go recommendation
- link the recommendation to the hosted environment used for review
